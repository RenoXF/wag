POST /connections/start

Start or create a new WhatsApp session.

Body Type Description
deviceId required string Unique device ID (alphanumeric, dash, underscore)
phoneNumber optional string Phone number for pairing code (instead of QR)
webhookUrl optional string URL for webhook notifications
name optional string Display name 
// Request
POST /connections/start
{
  "deviceId": "my-device",
  "phoneNumber": "6281234567890",
  "webhookUrl": "http://localhost:8000/callback/whatsapp/uuid",
  "name": "My WhatsApp"
}

// Response
{
  "success": true,
  "message": "Connection started successfully",
  "data": {
    "connectionState": "connecting",
    "sessionId": "my-device",
    "hasQrCode": true,
    "hasPairingCode": false
  }
}

POST /messages/send-text-message

Send a text message to a WhatsApp user or group.

Body Type Description

recipient required string Recipient JID (e.g., 628xxx@s.whatsapp.net, 12036xxx@g.us)

GET /logs/sse/:deviceId

Real-time log streaming via Server-Sent Events (30s timeout). 

const es = new EventSource('/logs/sse/my-device');
es.onmessage = (event) => {
  const { data } = JSON.parse(event.data);
  data.forEach(log => console.log(log));
};

Webhooks

WAG sends POST requests to your webhook URL with Content-Type: application/x-www-form-urlencoded body (qs.stringify({ event, data })).

POST /test-callback

Test if a webhook URL is reachable and returns 200.

Body Type Description
host required string Webhook URL to test 
POST /test-callback
{ "host": "http://localhost:8000/callback/whatsapp/xxx" }

// Response
{ "success": true, "message": "Callback URL is valid and reachable." }

Webhook Events

Event Data Description
ping { deviceId } Keepalive / reachability check
auth { auth: { via: "qr_code"|"pair_code", data } } QR code or pairing code received
connecting {} Device is connecting
ready {} Device connected + heartbeat (every 30 min)
state "open"|"connecting"|"close" Connection state changed
close { reason, isRestart } Connection closed
message_sent { id, deviceId, message, recipient } Message sent successfully
message_error { id, deviceId, message, recipient, error } Message failed to send

Webhook Headers

Header Description
Content-Type application/x-www-form-urlencoded
X-Session-ID Session ID that sent the event
X-Event Event name
X-Timestamp ISO timestamp

Webhook Body Format

// Received as URL-encoded form data
// event=ready&data=%5Bobject+Object%5D

// Parse with qs library:
const { event, data } = qs.parse(body);
